# 9a — secrets: TOTP + Watchtower

ROADMAP 9a: TOTP/2FA code generation + Watchtower (weak / reused / breached via HIBP k-anonymity).

## Audit (current state)
- Vault entries store typed `fields` encrypted as JSON (`routes/vault.py`); reveal decrypts them.
- `services/pwtools.py` has `estimate_strength` (score 0–4), card helpers. No TOTP, no Watchtower.
- `pyotp` is NOT installed → implement TOTP from RFC 6238 in stdlib (hmac/base64/struct), like web push.

## 9a-1 — TOTP + Watchtower (unittest) — `tests/test_vault_totp_watchtower.py`
Files: `services/pwtools.py`, `routes/vault.py`.
- TOTP: `totp_now(secret_b32, period=30, digits=6, t=None)` (RFC 6238) + `totp_remaining(period, t)`.
  A `totp` field on an entry holds the base32 secret. `GET /vault/{id}/totp` (vault-gated) → `{code, seconds}`;
  404 when the entry has no totp secret.
- Watchtower: `is_weak(pw)` (score ≤ 1), `find_reused(entries)` (group ids sharing a password),
  `breach_count(password, fetch)` (HIBP k-anonymity: SHA1 → send 5-char prefix → match suffix; injectable
  fetch). `GET /vault/watchtower` (vault-gated) decrypts all password entries → `{weak, reused, breached}`.
Tests (>=8): totp_rfc_vector, totp_same_window_stable, totp_remaining_range, totp_endpoint_returns_code,
totp_endpoint_no_secret_404, is_weak, find_reused, breach_count_parses, watchtower_weak, watchtower_reused,
watchtower_breached, watchtower_requires_unlock_403.
