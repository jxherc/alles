# 8b — calendar: invites & booking

ROADMAP 8b: invites + RSVP (local + email), video-meeting links, appointment/booking pages (1a).

## Audit (current state)
- Events have only a freeform `guests` text field — **no structured attendees, no RSVP**.
- `services/mail.send_mail(...)` exists (SMTP) → invite emails are possible (best-effort; needs a
  configured account, so tests mock it).
- `share.VALID_KINDS` includes `event` but `/s/{token}` doesn't render it; **no RSVP or booking page**.
- `/calendar/free` already computes open slots (reused for booking availability).
- No `meeting_url` on events.

## 8b-1 — invites + RSVP + video links (unittest) — `tests/test_calendar_invites.py`
Files: `core/database.py` (`EventAttendee` model + `CalendarEvent.meeting_url`), `routes/calendar.py`,
`routes/shared.py` (public RSVP page), `services/meet.py` (jitsi room helper).
- `EventAttendee`: id, event_id, name, email, status (invited|accepted|declined|tentative), token.
- `POST /calendar/{eid}/invite {name,email}` → attendee + token (best-effort email send, never fails create).
- `GET /calendar/{eid}/attendees`; `DELETE /calendar/attendees/{aid}`.
- `POST /rsvp/{token} {status}` (public) sets the attendee status; `GET /rsvp/{token}` renders a small page.
- `meeting_url` column + EventBody/EventPatch/_fmt; `services/meet.jitsi_url(title)` mints a room.
Tests (>=8): invite_creates_attendee, attendee_has_token, list_attendees, rsvp_sets_status,
rsvp_unknown_404, rsvp_bad_status_400, delete_attendee, meeting_url_roundtrip, jitsi_url_shape,
fmt_has_meeting_url, invite_email_best_effort.

## 8b-2 — booking pages (unittest) — `tests/test_booking.py`
Files: `core/database.py` (`BookingPage`), `routes/calendar.py`, `routes/shared.py` (public book page).
- `BookingPage`: id, token, title, duration_min, work_start, work_end, days_ahead, calendar_id.
- `POST/GET/DELETE /calendar/booking-pages`.
- `GET /book/{token}` → public page listing free slots over the next N days (reuse free-slot logic).
- `GET /book/{token}/slots?date=` → JSON free slots; `POST /book/{token}` {date,time,name,email} → creates a
  CalendarEvent (duration applied) + an accepted attendee; returns confirmation.
Tests (>=8): create_booking_page, page_has_token, slots_excludes_busy, book_creates_event,
book_sets_duration, book_adds_attendee, book_unknown_token_404, book_past_or_busy_rejected,
