# 11a — platform: macOS native

ROADMAP 11a: PhotoKit / EventKit / Keychain / iCloud Drive full integration (PyObjC where it pays off);
iCloud Drive watch-folder ingestion; back the vault onto Keychain. macOS-only, guarded — non-darwin must
fail loud.

## Audit
`services/macos_bridge.py` already has Keychain (`security` CLI) + EventKit (`icalBuddy`) seams, all
`_require_darwin`-guarded. `services/photo_sync.py` has the PhotoKit/osxphotos export seam. Gaps:
EventKit returns raw lines (no structured parse); no capability status; no iCloud Drive path/watch
helper; no route surface; non-darwin contract untested.

## 11a-1 — bridge expansion + routes (unittest) — `tests/test_macos_bridge.py`
Files: `services/macos_bridge.py`, new `routes/macos.py`, `app.py` (mount).
- `capabilities()` → `{platform, available, keychain, eventkit, photokit, icloud}` (available ==
  darwin; sub-flags reflect tool presence).
- `_parse_ical_output(text)` → `[{title, detail}]` (structured from icalBuddy's bulleted output);
  `export_calendar`/`export_reminders` return structured rows on darwin.

- `routes/macos.py`: `GET /api/macos/status` (capabilities), `POST /api/macos/calendar` +
  `/api/macos/reminders` (guarded → **503** off-darwin, structured rows on-darwin).
Tests (≥8, real on this Windows box — the non-darwin "fail loud" contract): is_mac_matches_platform,
require_darwin_raises_off_darwin, keychain_set_fails_loud, export_calendar_fails_loud,

parse_ical_multiple, icloud_dir_none_off_darwin, api_status_unavailable_off_darwin, api_calendar_503_off_darwin.
