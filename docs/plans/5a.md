# 5a — mail: triage

ROADMAP 5a: one-click List-Unsubscribe, saved/smart mailboxes, advanced search operators,
archive + mute thread. IMAP can't be exercised live, so everything is **cache-first + pure helpers**
(IMAP move/flag stays best-effort in the route); all tests run against the cache + helpers.

## 5a-1 — backend (unittest)
Files: `core/database.py`, `services/mail.py`, `services/mail_cache.py`, `routes/mail.py`.
- `mail.parse_list_unsubscribe(header)` → `{http, mailto}` (pure). `fetch_inbox` best-effort
  extracts the `List-Unsubscribe` header into each cached row.
- `CachedMessage.list_unsubscribe` + `muted` columns; `_to_msg` exposes them.
- `mail.parse_search_query(q)` → `{from, to, subject, has_attachment, before, after, text}` (pure).
- `mail_cache.advanced_search(db, account_id, spec, limit)` — applies from/subject/before/after/text
  on the cache (operators the cache can answer).
- `mail_cache.get_unified`/`get_filtered` exclude `muted`; `mail_cache.mute(db, aid, subject)` sets
  muted on all rows whose normalized subject matches; `mail_cache.archive(db, aid, folder, uid)`
  drops the cached row.
- `SavedSearch(mail_saved_searches)` model + `GET/POST/DELETE /api/mail/saved-searches`.
- Routes: `GET /api/mail/adv-search/{aid}?q=`, `POST /api/mail/mute/{aid}`,
  `POST /api/mail/archive/{aid}`.

Tests (`tests/test_mail_triage.py`, ≥12):
1. `parse_unsub_http` 2. `parse_unsub_mailto` 3. `parse_unsub_both` 4. `parse_query_operators`
5. `parse_query_plain_text` 6. `adv_search_from` 7. `adv_search_subject` 8. `adv_search_before_after`
9. `mute_hides_thread` 10. `archive_removes_from_cache` 11. `saved_search_crud`
12. `unified_excludes_muted` 13. `cached_row_has_list_unsubscribe`
