# 5c — mail: rich compose

ROADMAP 5c: HTML compose + inline images + multiple signatures. SMTP send stays best-effort; the
testable core is the MIME builder, the inline-image embedding, and signature CRUD.

## 5c-1 — backend (unittest)
Files: `services/mail.py`, `services/mail_compose.py` (new), `core/database.py`, `routes/mail.py`.
- `_build_message(..., html="", inline=None)` — when `html` given, `set_content(text)` +
  `add_alternative(html, subtype="html")`; each `inline` ({cid, data, subtype}) is added as a
  related part on the html so `cid:` refs resolve (multipart/related).
- `send_mail(..., html="", inline=None)` threads those through.
- `mail_compose.embed_inline(html, get_bytes)` — rewrite `/api/uploads/{id}` `<img>` srcs to
  `cid:{id}` and return the inline parts list (uses `get_bytes(id)` so it's testable without uploads).
- `ScheduledMail.html` column so scheduled/undoable HTML sends keep their html.
- Signatures in settings (`mail_signatures` = [{id, name, body}]); `GET/POST/DELETE
  /api/mail/signatures`.
- send / schedule / send-undoable bodies accept `html` (+ inline resolved from the html's upload
  refs in the route).

Tests (`tests/test_mail_compose.py`, ≥8):
1. `build_plain_only` 2. `build_html_alternative` 3. `build_inline_image_has_cid`
4. `embed_inline_rewrites_src` 5. `embed_inline_no_images` 6. `embed_inline_collects_bytes`
7. `signatures_crud` 8. `signature_update_by_id` 9. `scheduled_carries_html`

## 5c-2 — frontend (Playwright)
Files: `static/js/mail.js`, `static/style.css`, stamps. (Compose is a local form — only a seeded
account is needed, no IMAP.)
- A light HTML toolbar (bold / italic / link) over a `contenteditable` body.
- An inline-image button → upload via `/api/uploads` → insert `<img>`.
- A signature picker (dropdown) that appends the chosen signature; manage signatures.

Tests (`tests/pw_mail_compose_5c.py`, ≥8):
1. `compose_has_richbar` 2. `bold_wraps_selection` 3. `signature_picker_present`
4. `signature_appends` 5. `image_btn_inserts_img` 6. `add_signature` 7. `html_sent_or_queued`
8. `zero_console_errors`
