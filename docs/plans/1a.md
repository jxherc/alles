# 1a — Share & Publish primitive

Audit: `docs/evidence/1a/` (findings.md, curl-share.txt, curl-gap-probes.txt, pw_audit.txt, view-*.png).
Only `Session` shares today; everything else 404/405. Build a generic primitive (don't touch the
working session-share path — keep it for back-compat and fall back to it in the viewer).

## 1a-1 — generic share backend  (unittest, ≥12, RED→GREEN)
- **`Share` model** (`core/database.py`): `token` (unique, hex), `kind`
  (doc|file|photo|album|contact|event|session), `ref` (resource id or vault/file path),
  `level` (view|download), `created_at`. `create_all` makes it.
- **`services/share.py`**: `mint(db, kind, ref, level)` (idempotent per (kind,ref); level upgradable),
  `lookup(db, token)`, `token_for(db, kind, ref)`, `revoke(db, token)`, `revoke_ref(db, kind, ref)`,
  `md_to_html(text)` (dependency-free minimal renderer: headings, bold/italic, inline+fenced code,
  links, ul/ol, paragraphs, HTML-escaped).
- **`routes/shared.py`**: `POST /api/share {kind,ref,level?}` → `{token,url,kind,ref,level}` (400 bad
  kind / empty ref); `GET /api/share?kind=&ref=` → `{token|null,url|null}`; `DELETE /api/share
  {kind,ref}` → `{ok}`. Enhanced `GET /s/{token}`: resolve `Share` first — doc→`vault_md.read`→
  `md_to_html` in the read-only shell; file→`files_store.abspath` streamed (inline, or attachment when
  `level=download`); photo→`Photo` row → image bytes; else fall back to the existing `Session.share_token`.
- Tests (`tests/test_share.py`): mint_idempotent, mint_diff_ref_diff_token, lookup, token_for_none_then_set,
  revoke_by_token, revoke_ref, bad_kind_raises, empty_ref_raises, level_default_view_and_download,
  md_to_html_rules (heading/bold/code/link/escape), api_post_mint, api_post_bad_kind_400,
  api_get_state, api_delete_revokes, view_doc_html, view_file_inline, view_file_download,
  view_photo_image, view_unknown_404, view_revoked_404, session_fallback_still_works.

## 1a-2 — share UI: docs publish + files share  (Playwright, ≥8, RED→GREEN)
- **`static/js/util.js`** `shareResource(kind, ref, level)` — POST `/api/share`, copy `origin+url`,
  toast (reuse existing toast); `unshareResource(kind, ref)` — DELETE; `shareState(kind, ref)` — GET.
- **docs** (`vaultmd.js` + editor head): a **Publish** button on an open doc → mints `kind=doc,
  ref=path`, copies link, shows shared state; click again → unshare.
- **files** (`files.js` row actions): a **share** affordance per file → mints `kind=file, ref=path`,
  copies link.
- Bump `?v` / `_v` / `sw.js VERSION` so the new JS reaches loaded clients.
- Verify (`tests/pw_share_1a.py`): docs_publish_button_present, docs_publish_mints_and_copies,
  docs_shared_state_shows, docs_unpublish_revokes, files_share_action_present, files_share_mints,
  public_doc_link_opens_readonly, zero_console_errors.
