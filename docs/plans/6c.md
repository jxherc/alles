# 6c — files: share

ROADMAP 6c: file/folder share links + permission levels (1a) + file comments.
File share + view/download already shipped (1a). Build folder share + file comments only.

## 6c-1 — backend (unittest)
Files: `services/share.py`, `routes/shared.py`, `routes/files.py`, `core/database.py`.
- Allow `folder` in `share.VALID_KINDS`.
- `GET /s/{token}` (kind=folder) -> read-only HTML index listing every file under the folder,
  each linking to `/s/{token}/<relpath>`.
- `GET /s/{token}/{subpath:path}` -> serve a file inside the shared folder, confined to it
  (path-traversal safe), inline or attachment per `level`.
- `FileComment` model (path, body, author, parent_id, resolved, created_at) — new table.
  `GET/POST /api/files/comments`, `POST /api/files/comments/{id}/resolve`,
  `DELETE /api/files/comments/{id}` (mirror DocComment). `/list` decoration carries a `comments` count.
