# 8d — CardDAV sync

ROADMAP 8d: two-way contact sync (reuse the caldav pattern) for iCloud/Google with the user's creds.

## Audit (current state)
- `services/caldav_sync.py` is the template: lazy/defensive, config in `data/caldav.json`, `status/sync`,
  pull-by-UID + push-local-only. No CardDAV equivalent exists.
- `services/vcard.py` has `parse_vcards(text)` + `to_vcard(contacts)` (reused).
- `Contact` has no CardDAV sync columns.

## 8d-1 — carddav_sync (unittest) — `tests/test_carddav.py`
Files: `core/database.py` (`Contact.carddav_uid/href/etag`), `services/carddav_sync.py`,
`routes/carddav.py`, `app.py` (register router).
- Pure/testable core with an **injectable client** so tests need no network:
  `parse_report(xml)` → `[{href,etag,vcard}]`; `vcard_uid(text)`; `build_vcard(contact, uid)`.
  `sync(client, db)`: pull remote (match local by `carddav_uid`, update/create) + push local-only
  (`client.put(uid,text)` → href, stamp `carddav_uid/href`). Returns `{pulled, pushed}`.
- Real `DavClient` (httpx PROPFIND/REPORT/PUT) used when no client injected; `status/connect/sync` endpoints
  mirror caldav; not-configured → `{error}`.
Tests (>=8): parse_report_extracts, vcard_uid, build_vcard_has_fn_uid, sync_pull_creates,
sync_pull_updates, sync_push_local_only, sync_idempotent, not_configured_error, status_shape, push_sets_uid.

## 8d-2 — frontend (Playwright) — `tests/pw_carddav_8d.py`
Files: `static/js/contacts.js`, `static/index.html`, `static/style.css`, stamps.
- A "CardDAV" button in the contacts header → a connect dialog (url/username/password) + a sync action that
  shows pulled/pushed counts; status reflects connected.
Tests (>=8): carddav_button, connect_dialog_opens, connect_saves_status, sync_runs, sync_shows_counts,
status_connected_after, disconnect_clears, zero_console_errors. (sync uses a stubbed in-memory feed via a
test-only `?fake=1` flag so no real server is needed.)
