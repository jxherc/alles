# 10b — aide: background agents (durable detached runs)

ROADMAP 10b: durable detached runs that survive browser close; status in the runs drawer.

## Audit (`docs/evidence/10b/findings.md`)
`POST /api/agent/background` (routes/chat.py:436) already detaches via `asyncio.create_task` and
survives tab close; `agent_state.record_event` persists `events[]` to JSON on every event; `get_run`/
`find_active_run` read from disk. **Gaps:** (1) the assistant's prose text isn't persisted
incrementally (only events are) so a reconnecting client can't show the answer-so-far; (2) there's no
tail endpoint to follow a run's progress after reconnect; (3) the frontend never reattaches to a live
run on reload. `GET /api/agent/runs/active` exists but no JS calls it.

## 10b-1 — durable run state + reconnect tail (unittest) — `tests/test_agent_background.py`
Files: `services/agent_state.py` (add `text` to the run state), `services/agent_runtime.py` (persist
accumulated prose per turn + at finish), `routes/agent.py` (`GET /agent/runs/{id}/events?since=N`).
- Run JSON carries `text` (accumulated assistant prose), updated each turn so a reconnect shows progress.
- `GET /api/agent/runs/{run_id}/events?since=N` → `{events: events[N:], next, status, text, turn, done}` —
  a durable, pollable tail (no live-SSE coupling), 404 on unknown run.
- `GET /api/agent/runs/active?session_id=` (exists) is the reconnect entry point.
Tests (≥8): start_run_has_text_field, update_run_text_persists, run_persists_text_incrementally
(runtime-driven), events_since_zero_returns_all, events_since_n_returns_new, events_cursor_advances,
events_includes_status_text_done, events_unknown_404, active_running_then_empty,
background_requires_session_404.

## 10b-2 — frontend reattach + live drawer (Playwright) — `tests/pw_agent_bg_10b.py`
Files: `static/js/chat.js` (or aide bootstrap), `static/js/runs.js`, stamps.
- On aide view load (or session open), call `/api/agent/runs/active?session_id=`; if a run is live,

Tests (≥8): reattach_finds_live_run, reattach_renders_progress, reattach_indicator_shows,
