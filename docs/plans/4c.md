# 4c — money: forecast & investments (Simplifi)

ROADMAP 4c: spending-plan forecast, net-worth history, watchlists + alerts + customizable
dashboard, investment holdings + cost basis.

## 4c-1 — backend (unittest)
Files: `core/database.py`, `routes/money.py`.

- `Holding(money_holdings)`: id, symbol, qty, cost_basis (per share), price, name, created_at.
- `Watch(money_watches)`: id, kind (payee|category), value, created_at.
- `GET /api/money/forecast?month=&as_of=YYYY-MM-DD` → `{start_balance, projected, recurring:[...],
  line:[{date, balance}]}`. start = total net balance as of `as_of` (default today); project each
  active recurring occurrence from > as_of through month-end via `_advance`.
- `GET /api/money/networth-history?months=N&as_of=` → `[{month, net_worth}]` (net worth at each
  month-end, oldest→newest).
- Holdings CRUD: `GET/POST/PATCH/DELETE /api/money/holdings`; each row → `value = qty*price`,
  `cost = qty*cost_basis`, `gain = value-cost`, `gain_pct`. GET returns rows + totals.
- `Watch` CRUD: `GET/POST/DELETE /api/money/watches`.
- `GET /api/money/alerts?month=&big=` → `{large_purchases:[...], upcoming_bills:[...],
  watch_hits:[...]}` (large = |amount| ≥ big this month; upcoming = recurring due ≤7d; watch_hits =
  txns this month matching a watch payee/category).
