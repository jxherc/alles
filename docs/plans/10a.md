# 10a — aide: code intelligence

ROADMAP 10a: semantic codebase index (reuse 1c), tool-event hooks (run a rule on
write_file/edit_file/etc.), git worktree isolation (per-run worktree for parallel agents).

## Audit
See `docs/evidence/10a/findings.md`. 1c `services/textindex.py` is the reuse target; the run loop's
per-tool `record_event` (~agent_runtime.py:553) is the single hook choke point; `agent_cwd` (contextvar)
redirects all file/shell ops, so a worktree just needs `agent_cwd` repointed after `start_run`.

## 10a-1 — semantic codebase index (unittest) — `tests/test_codeindex.py`
Files: new `services/codeindex.py`, `routes/code.py` (router), `app.py` (mount), `services/agent_tools.py`
(`search_code` tool).
- `services/codeindex.py`: `CODE_EXTS` allow-list; `iter_code_files(root)` (skips .git, node_modules,
  __pycache__, data/, dotdirs, binaries, >256KB); `reindex(db, root)` → `textindex.reindex_kind(db,
  "code", items)`; `search(db, query, k=8)` → `textindex.search(db, query, "code", k)`; `stats(db)`.
- `GET /api/code/search?q=&k=` and `POST /api/code/reindex` (indexes the agent cwd / repo root).
- Agent tool `search_code(query)` returns ranked file/chunk hits so the agent can find code by meaning.
Tests (≥8): exts_filter, iter_skips_vcs_and_deps, iter_skips_large, reindex_counts, search_finds_by_meaning,
search_scoped_to_code_kind, search_empty_index, stats_after_reindex, api_search_shape, api_reindex,
tool_search_code_returns_hits.

## 10a-2 — tool-event hooks (unittest) — `tests/test_agent_hooks.py`
Files: `services/automations.py` (`agent_tool` trigger + `on_agent_tool(name,args,result,run_id)`),
`services/agent_runtime.py` (call `fire_tool_hooks` at the choke point).
- `AutomationRule.trigger == "agent_tool"`, `trigger_arg` = a tool-name glob (e.g. `write_*`, `*`).
  `on_agent_tool(tool, args, result, run_id)` matches enabled rules whose glob matches `tool` and runs
  their action (reuse the existing action runner: create_task/create_note/push).
- `run_agent` calls `automations.on_agent_tool(name, args, result, run_id)` right after `record_event`.
Tests (≥8): glob_exact_match, glob_wildcard, glob_no_match, disabled_rule_skipped, fires_create_task,
fires_on_write_file_only (glob `write_*`), non_agent_tool_rules_untouched, multiple_rules_fire,
hook_called_from_runtime (drive run_agent, assert action ran), hook_safe_when_no_rules.

## 10a-3 — git worktree isolation (unittest) — `tests/test_agent_worktree.py`
Files: new `services/worktrees.py` (`setup`/`teardown`/`is_git_repo`), `services/agent_runtime.py`
(gate on `settings["agent_worktree"]`), `routes/agent.py` (none) — settings only.
- `services/worktrees.py`: `is_git_repo(path)`, `setup(root, run_id) -> path|None` (runs
  `git worktree add --detach <tmp> HEAD`), `teardown(root, path)` (runs `git worktree remove --force`).
- `run_agent`: when `settings.get("agent_worktree")` and the cwd is a git repo, create a worktree,
  repoint `settings["agent_cwd"]` to it, record `worktree` on the run, and tear down in `finally`.
Tests (≥8): is_git_repo_true, is_git_repo_false, setup_creates_worktree, setup_has_repo_files,
setup_non_repo_returns_none, teardown_removes_worktree, edits_in_worktree_isolated (write in wt doesn't
touch root), runtime_repoints_cwd (drive run_agent with agent_worktree, assert run.cwd == worktree),
runtime_teardown_after_run, runtime_skips_when_not_repo.
