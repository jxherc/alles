# 10c — aide: plugins via skills (git-backed, shareable)

ROADMAP 10c: install/update a skill bundle from a git URL; export your own (self-hosted
marketplace-lite).

## Audit
Skills already exist: `services/skills_store.py` (CRUD over `data/skills/<slug>/SKILL.md`),
`services/skills_github.py` `import_from_github(url)` (repo/tree/blob → import), `routes/skills.py`
(list/match/catalog/install/import-github/upload/CRUD), `static/js/skills.js`. **Gaps for 10c:**
(1) imported skills don't record their **source** URL, so they can't be updated; (2) no **update**
(re-pull from source); (3) no **export** of your own skill to share.

## 10c-1 — source tracking + update + export (unittest) — `tests/test_skills_marketplace.py`
Files: `services/skills_store.py`, `services/skills_github.py`, `routes/skills.py`.
- `_serialize`/`_parse`/`upsert_skill`/`get_skill`/`list_skills` carry an optional `source:` line.
- `import_from_github` stamps each imported skill's `source` (per-skill `…/blob/<branch>/<path>` for
  repo/tree imports; the input url for a direct file).
- `skills_store.export_md(slug)` returns the full SKILL.md; `export_all()` → `[{slug, md}]`.
- `update_from_source(slug)` re-imports from the stored source (no-op if none).
- Endpoints: `POST /api/skills/{slug}/update`, `GET /api/skills/{slug}/export` (raw markdown download).
Tests (≥8): source_roundtrips, serialize_includes_source, upsert_without_source_omits_line,
export_md_returns_full, export_unknown_none, export_all_bundle, import_sets_source (mocked fetch),
update_re_fetches (mocked), update_no_source_noop, api_export_markdown, api_update.
