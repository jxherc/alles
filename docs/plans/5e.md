# 5e — mail: labels & push

ROADMAP 5e: multi-label per message + category tabs, IMAP IDLE push. Labels + the category
heuristic are cache-side and fully tested; IMAP IDLE is best-effort (needs a live server) — the
testable part is the heuristic + a "live" poll toggle.

## 5e-1 — backend (unittest)
Files: `core/database.py`, `services/mail.py`, `services/mail_cache.py`, `services/mail_idle.py`
(new, best-effort), `routes/mail.py`.
- `CachedMessage.labels` (csv) column; `_to_msg` exposes a `labels` list.
- `mail_cache.set_labels(db, aid, folder, uid, labels)`, `add_label`, `by_label(db, aid, label)`.
- `mail.categorize(sender, subject, list_unsubscribe)` → primary | social | promotions | updates
  (pure heuristic: list-unsubscribe / promo words → promotions; social domains → social; no-reply /
  notification → updates; else primary).
- `mail_cache.by_category(db, aid, cat)` — cache filtered by the computed category.
- Routes: `POST /api/mail/labels/{aid}` (set a message's labels), `GET /api/mail/by-label/{aid}`,
  `GET /api/mail/category/{aid}?cat=`.
- `mail_idle.idle_available(acct)` + best-effort `idle_wait` (untested; real IDLE needs a server);
  a `mail_live` settings flag for a faster poll.

Tests (`tests/test_mail_labels.py`, ≥10):
1. `set_labels` 2. `labels_normalized` 3. `add_label_keeps_existing` 4. `by_label_filters`
5. `to_msg_labels_list` 6. `cat_promotions_unsubscribe` 7. `cat_social_domain` 8. `cat_updates_noreply`
9. `cat_primary_default` 10. `by_category_filters` 11. `cat_route`
