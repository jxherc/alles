# 3e — Inline comments

ROADMAP 3e: threaded comments/discussions anchored to text in a doc.

## Design
- `DocComment(Base)` — id, doc (vault path), anchor (quoted text), body, author (default "me"),
  parent_id (null = thread root; set = reply), resolved (bool), created_at. New table → `create_all`.
- A thread = a root comment + its replies (ordered by created_at). Anchor is the quoted span; the
  frontend highlights the first occurrence of the anchor text in the rendered note, and shows
  threads whose anchor no longer matches as "orphaned".

## 3e-1 — backend (unittest)
Files: `core/database.py`, `routes/vault_md.py`.
- `POST /api/vault-md/comments {path, anchor, body, author?}` → create a thread root.
- `POST /api/vault-md/comments {parent_id, body, author?}` → reply (inherits doc+anchor from root).
- `GET /api/vault-md/comments?path=` → `{threads:[{...root, replies:[...], resolved, orphaned}]}`,
  newest thread last (or sorted by created_at), `orphaned` true when anchor text not in the note.
- `POST /api/vault-md/comments/{id}/resolve` → toggle (resolves the whole thread via the root).
- `DELETE /api/vault-md/comments/{id}` → delete a comment; deleting a root deletes its replies.

## 3e-2 — frontend (Playwright)
Files: `static/index.html`, `static/js/vaultmd.js`, `static/style.css`, version stamps.
- `comments` toolbar button → `#wiki-comments` panel listing threads (anchor quote, body, replies,
  resolve/reply/delete). Badge count of open threads.
- Select text in `#wiki-preview` → a floating "💬 comment" button → prompts for the comment body →
  creates a thread anchored to the selection; the anchored span highlights inline.
- Reply inline; resolve hides/greys the thread; delete removes it.
