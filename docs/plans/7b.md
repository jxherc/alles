# 7b — gallery: places & memories

ROADMAP 7b: GPS map view (Leaflet/OSM, no API key) + date-based Memories / PIL collages (no ML).

## Audit (what already exists — do NOT rebuild)
- GPS EXIF → decimal lat/lon is already extracted (`photos_store._gps_to_decimal`, surfaced in
  `_fmt`'s `exif.lat/lon`). The lightbox already shows a per-photo OpenStreetMap **link**.
- No **map view** (clustered markers over the library). Genuine gap.
- No **memories** ("on this day" across prior years) and no **collage** generation. Genuine gaps.
- `static/vendor/` holds only `cm6.bundle.js`; Leaflet is not vendored. OSM tiles are reachable with
  a normal browser UA; `unpkg` is reachable for vendoring `leaflet.js`/`leaflet.css`.

## 7b-1 — backend (unittest) — `tests/test_photos_places.py`
Files: `routes/photos.py`, `services/photos_store.py`.
- `GET /api/photos/map` → `{points:[{id,lat,lon,thumb,original,caption,taken_at}], count}`; only
  photos with GPS in EXIF; excludes hidden + deleted.
- `GET /api/photos/memories?date=YYYY-MM-DD` (default today) → `{groups:[{years_ago,year,date,items}], count}`
  where each group is photos taken on the same month/day in a strictly earlier year; excludes hidden +
  deleted; groups sorted most-recent-year first.
- `POST /api/photos/collage {ids:[...], cols?}` → builds a PIL grid collage from those photos and saves
  it as a new photo (via `import_image`); returns the new photo `_fmt`. `photos_store.make_collage(paths,
  cols, cell)` does the pure image work. Empty ids → 400; unknown ids skipped; <1 usable → 400.
