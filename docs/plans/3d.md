# 3d — Workspace AI + DB extras

ROADMAP 3d: ask-anything over the vault (1c), in-doc automations, charts on query/base, form
blocks, web-clipper bookmarklet. Backend for ask/clip/automation already shipped; this microversion
fixes the bookmarklet, adds forms + charts, and builds the docs-side UI.

## 3d-1 — backend (unittest)
Files: `routes/vault_md.py`, `services/vault_md.py`.

- `clipper-bookmarklet` derives its origin from the request (no more `YOUR_ALLES_ORIGIN`).
- `POST /form-submit {path?, target, fields?, values}` → `vault_md.append_form_row(target, values,
  fields)` appends a markdown table row (creates the header from field/value keys on first submit),
  snapshots the target first. Returns `{ok, path, count}`.
- `parse_query_spec` captures `chart: bar|pie|line`; `query_block` passes `chart` through and, when
  grouped, returns a per-group `count` so the chart can be drawn server-data-driven.
- Lock in the existing `/ask` (sources, empty-q, ranking, reindex-if-empty), `/clip` (note created,
  title sanitised, source line), and base-cell → `on_doc_saved` doc_tag automation (fires once per
  doc, creates a task).

## 3d-2 — frontend (Playwright)
Files: `static/index.html`, `static/js/vaultmd.js`, `static/style.css`, version stamps.

- `ask` toolbar button → `#wiki-ask` panel: input + ask button + ranked source list; clicking a
  source opens that note. Calls `/api/vault-md/ask`.
- Web-clipper section inside the ask panel: shows the draggable bookmarklet (`/clipper-bookmarklet`).
- Charts: ```query``` / ```dataview``` fence with `chart: bar|pie|line` + `group:` → inline SVG
  chart rendered from the grouped counts (alongside/!instead of the list).
- Form blocks: ```form``` fence (`target:` + `fields: a, b, c`) → rendered inputs + submit; submit
  POSTs `/form-submit` and toasts; the target note gains a row.

Tests (`tests/pw_workspace_3d.py`, ≥8):

4. `clipper_bookmarklet_shown` 5. `chart_renders_svg` 6. `form_block_renders_inputs`
7. `form_submit_appends_row` 8. `zero_console_errors`
