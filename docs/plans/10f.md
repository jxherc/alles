# 10f — aide: realtime voice (gated)

ROADMAP 10f: full-duplex (barge-in) voice, **gated** on wiring a realtime-capable provider into the
endpoint model. Ships only when a real backend exists — no fake shell. Otherwise the feature stays
hidden behind the gate.

## Audit
`routes/voice.py` has STT/TTS/audio-overview (turn-based, not full-duplex). `ModelEndpoint.models_list()`
lists chat models; `services/llm.detect_provider` routes wire protocols. No realtime concept. A realtime
provider is signalled by an enabled endpoint exposing a model whose id matches /realtime/ (OpenAI
Realtime convention, e.g. `gpt-4o-realtime-preview`). With none configured the feature is gated off.

## 10f-1 — realtime gate + status + session handoff (unittest) — `tests/test_realtime_gate.py`
Files: new `services/realtime.py`, `routes/voice.py`.
- `services/realtime.py`: `find_realtime_endpoint(db) -> (ep|None, model|"")` (first ENABLED endpoint
  with a /realtime/i model); `status(db) -> {available, reason, model}`.
- `GET /api/voice/realtime/status` → the status dict. `POST /api/voice/realtime/session` → **503** with a
  reason when gated (no provider); when available, returns the real connection descriptor
  `{provider, base_url, model}` the client needs (no fabricated stream).
Tests (≥8): no_endpoints_unavailable, non_realtime_model_unavailable, realtime_model_available,
disabled_endpoint_ignored, case_insensitive_match, find_returns_ep_model, status_endpoint_shape,
session_gated_503, session_available_descriptor, reason_present_when_unavailable.
