# 9b — secrets: attachments + per-item share

ROADMAP 9b: encrypted file attachments (AES-GCM blobs linked to an entry) + per-item share link (1a,
careful reveal scoping).

## Audit (current state)
- `services/crypto.py` encrypts/decrypts strings (AES-256-GCM, PBKDF2 from the master pw). No byte API.
- Vault has no attachments and no per-item share.

## 9b-1 — encrypted attachments (unittest) — `tests/test_vault_attachments.py`
Files: `services/crypto.py` (`encrypt_bytes`/`decrypt_bytes`), `core/database.py` (`VaultAttachment`),
`routes/vault.py`.
- Attachment blobs stored encrypted at rest under `<data>/vault_attachments/{id}.enc`.
- `POST /vault/{id}/attachments` (upload, vault-gated, encrypts with the master pw),
  `GET /vault/{id}/attachments` (list: id/name/size), `GET /vault/attachments/{aid}` (decrypt + download),
  `DELETE /vault/attachments/{aid}`.
Tests (>=8): encrypt_bytes_roundtrip, upload_creates, list_attachments, download_decrypts,
download_wrong_pw_fails, delete_removes, blob_is_ciphertext_on_disk, requires_unlock_403, size_recorded.

## 9b-2 — per-item share (unittest) — `tests/test_vault_share.py`
Files: `core/database.py` (`VaultShare`), `routes/vault.py`, `routes/shared.py` (public viewer).
- Mint (vault-gated): pick the revealable fields, encrypt them with a **fresh random key** (AES-GCM, raw
  key — NOT the master pw); store only the ciphertext + token; return `{token, key}` so the UI puts the key
  in the URL **fragment** (server never sees it). `GET /sv/{token}/data` (public) returns the ciphertext;
  `GET /sv/{token}` (public) is an HTML page that reads the key from `location.hash` and decrypts in the
  browser (WebCrypto). `DELETE /vault/{id}/share` revokes.
Tests (>=8): mint_returns_token_key, data_is_ciphertext, decrypt_with_key_roundtrips (Python AESGCM),
data_unknown_404, revoke_then_404, mint_requires_unlock_403, key_not_stored_server_side,
viewer_page_served, mint_idempotent_same_entry.

## 9b-3 — frontend (Playwright) — `tests/pw_vault_share_9b.py`
Files: `static/js/vault.js`, `static/index.html`, `static/style.css`, stamps.
- Entry form: attachments list + upload + download/remove.
- A "share item" action mints a link (`/sv/{token}#key`) and copies it; the public page decrypts + shows the
  item read-only.
Tests (>=8): attach_upload_shows, attach_download, attach_remove, share_item_button, share_mints_link,
public_share_decrypts, public_share_revoked_gone, zero_console_errors.
