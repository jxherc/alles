# 4a вАФ money: transaction depth

ROADMAP 4a: split a transaction across categories; structured tags + receipt attachments;
cleared/uncleared + reconcile-to-statement.

## 4a-1 вАФ backend (unittest)
Files: `core/database.py`, `routes/money.py`.

- `TxnSplit(money_txn_splits)`: id, txn_id (FK cascade), category, amount (positive magnitude).
- `Transaction` gains `tags` (csv), `receipt_id` (Upload id), `cleared` (bool) via `_add_col`.
- `GET /api/money/transactions/{tid}/splits` вЖТ `{splits:[{category, amount}]}`.
- `PUT /api/money/transactions/{tid}/splits {splits:[...]}` вЖТ replace; reject sum > |txn.amount| (400).
- `summary` honors splits: a split expense distributes its slices into `by_cat`, any uncovered
  remainder falls to the txn's own category. Unsplit txns unchanged.
- Tags normalized (lowercased/trimmed/deduped) on create + PATCH; `GET /transactions?tag=` filter.
- PATCH whitelist extended: `tags`, `receipt_id`, `cleared`.
- `GET /api/money/accounts/{aid}/reconcile?statement=` вЖТ `{cleared_balance, statement, difference,
  reconciled}` (cleared_balance = opening + sum of cleared txns).
