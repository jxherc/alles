# 1b — Offline & sync engine

Audit: `docs/evidence/1b/`. Today the SW does GET-shell caching only; mutating `/api/*` offline is
silently lost. Build a service-worker write-queue (IndexedDB outbox) + replay-on-reconnect + a pending
indicator. Don't touch the working GET-cache / push paths.

## 1b-1 — SW outbox + replay + pending indicator  (Playwright, ≥9, RED→GREEN)
- `static/sw.js`: intercept mutating `/api/*` (POST/PUT/PATCH/DELETE) — skip `/api/auth`, `/api/chat`,
  `/api/agent`, and `multipart/form-data`. Try network; on failure queue the serialized request in
  IndexedDB `alles-sync/outbox` and return synthetic `200 {queued:true}`. `flush()` replays FIFO,
  dropping each on a 2xx or a permanent 4xx, stopping on network error. `message` handlers:
  `alles-flush`, `alles-pending`. `notifyClients()` posts `{type:'alles-sync', pending:n}`.
- `static/js/sync.js` `initSync()`: register SW; on SW `alles-sync` message update `#sync-indicator`;
  on `online`/load post flush + pending. `app.js` calls it after `registerServiceWorker()`.
- `static/index.html`: `#sync-indicator` chip + CSS; bump `?v`/`_v`/SW VERSION.
- Verify (`tests/pw_sync_1b.py`): sw_controls, offline_write_returns_queued, outbox_has_entry,
  indicator_shows_pending, queue_survives_reload, online_replays_drains_outbox, server_received_write,
  indicator_clears, zero_console_errors.
