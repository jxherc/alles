# 5b — mail: send control

ROADMAP 5b: schedule send, undo send, snooze (jobs). SMTP send stays best-effort; all logic is
testable via a `process_due(db, now)` function and cache fields (no live SMTP/IMAP needed).

## 5b-1 — backend (unittest)
Files: `core/database.py`, `services/mail_outbox.py` (new), `routes/mail.py`, `services/jobs.py`.
- `ScheduledMail(mail_scheduled)`: id, account_id, to, cc, bcc, subject, body, in_reply_to,
  references, send_at (ISO), status (scheduled|sent|canceled), created_at.
- `mail_outbox.process_due(db, now_iso)` — sends each scheduled mail with `send_at <= now` and
  status scheduled (SMTP best-effort, never raises), marks `sent`; returns count. Registered as a
  30s job.
- Routes: `POST /api/mail/schedule/{aid}` (to/subject/body/send_at…); `GET /api/mail/scheduled`
  (status=scheduled, soonest first); `POST /api/mail/scheduled/{sid}/cancel` (→ canceled = undo);
  `POST /api/mail/send-undoable/{aid}` (schedules now + `delay` seconds, returns id for the undo
  toast).
- `CachedMessage.snoozed_until` (ISO) column; `mail_cache` get/get_unified/get_filtered hide a row
  while `snoozed_until > now`; `mail_cache.snooze(db, aid, folder, uid, until)`; routes
  `POST /api/mail/snooze/{aid}` and `GET /api/mail/snoozed/{aid}`.

Tests (`tests/test_mail_outbox.py`, ≥10):
1. `schedule_creates` 2. `scheduled_list_pending` 3. `cancel_scheduled` (undo)
4. `process_due_sends_past` 5. `process_due_skips_future` 6. `process_due_skips_canceled`
7. `send_undoable_schedules_future` 8. `snooze_hides_until_future` 9. `snooze_past_visible`
10. `snoozed_reappears_after_time` 11. `snoozed_list` 12. `unified_excludes_snoozed`
