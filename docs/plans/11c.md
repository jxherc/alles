# 11c — Unification: alles↔aide cross-app audit + polish

ROADMAP 11c (S). Acceptance: a Playwright sweep cross-navigates all subdomains with one login,
0 console errors; every subdomain scope + SSO handoff + cross-jump verified.

## Audit
The auth/SSO machinery is already correct (core/auth.py, routes/auth.py, app.py
TokenAuthMiddleware, static/js/subdomain.js). Existing unit tests cover handoff single-use /
expiry / bad-code (test_auth.py) and password/config (test_api_auth.py). Gaps are *integration*
coverage, which 11c adds to harden as the suite grows:
- no HTTP test of the full login→handoff→redeem→session round-trip
- no test of /api/auth/me shape with AUTH_ENABLED on (authed + unauthed)
- no test that the middleware actually gates /api/ (except /api/auth/*) when enabled
- no test of the localhost host-only cookie (no Domain=) vs a real domain
- no frontend test that every subdomain scopes correctly and cross-jumps

## Tasks

### 11c-1 — backend SSO handoff + auth scope (tests/test_sso_11c.py)
Drive the real app with AUTH_ENABLED toggled per-test (env var read at call time; isolate
settings to a tempfile). Tests: me_disabled_shape, me_enabled_unauthed, login_then_me_authed,
handoff_requires_auth, full_handoff_redeem_roundtrip, handoff_single_use, redeem_bad_code,
redeem_expired_code, middleware_gates_api, auth_routes_exempt, logout_revokes,
cookie_host_only_on_localhost.

### 11c-2 — frontend cross-app sweep (tests/pw_unify_11c.py)
One server, walk every SUBDOMAIN_VIEWS host (apex + aide + 14 apps). For each: load, assert it
boots (.app visible, not login-mode), correct body scope class (is-hub / is-aide / is-subapp),
the app crumb / primary view present, 0 console errors. Then verify cross-jump: from an app,
clicking the brand/crumb navigates back toward the hub. Assertions: apex_is_hub, aide_is_aide,
subapps_are_subapp, every_host_boots, every_host_zero_errors, primary_view_renders,
crumb_present_on_subapps, crossjump_back_to_hub, sidebar_scoped_to_aide, total_zero_errors.
